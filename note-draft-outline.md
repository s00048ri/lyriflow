# note記事構成案：LyriFlow — Claude Codeで歌詞テレプロンプターを作った話

- 「歌詞が覚えられないボーカリストが、Claude Codeで自分専用テレプロンプターを作った話」
- 「AIと一緒に1日で楽器アプリを作る — Claude Code × iPad × Cloudflare Pages」
- 「既存アプリに不満があったので、Claude Codeに相談したら1日でWebアプリが完成した」

## 構成

### 1. 導入：歌詞が覚えられない問題（〜800字）
- 自己紹介（バンドのボーカリストであること）
- ステージで歌詞を見ながら歌うニーズ
- iPadとBluetoothペダルを使ったテレプロンプター運用

### 2. 既存アプリへの不満（〜600字）
- Setlists by Bombing Brain：曲間でリスト画面に戻る、クラッシュ頻発
- OnSong：サブスク、機能過多
- LivePrompter：スクロール方式、外部エディタ必要
- 本当に欲しかったもの：「ペダルを踏み続けるだけでセットリスト全曲を通せる」シンプルなアプリ

### 3. Claude Codeに相談してみた（〜1200字）
- Claude Codeとは何か（簡潔に）
- プロジェクトブリーフを書いてClaude Codeに渡した流れ
- 最初の対話：「単一HTMLファイルでプロトタイプを作りたい」
- 驚きのポイント：要件を伝えたら動くプロトタイプが一発で出てきた
- 対話例（実際のやりとりを一部抜粋）

### 4. 視認性の改善 — 対話で磨く（〜1000字）
- 最初のプロトタイプの問題点（フォントサイズ、折り返し、次スライドプレビュー）
- 「こうしたい」を伝えたら即座に反映される体験
- フォントの自動フィット（CJK文字幅を考慮したロジック）
- 編集モードのスライドプレビュー追加

### 5. iPadで動かしたい — デプロイの試行錯誤（〜1500字）
- 最初の方針：ローカルHTTPサーバー → 同じネットワークにいない問題
- GitHub Pages → 社内ネットワークでGitHubがブロックされている問題
- Netlify Drop vs Cloudflare Pages の比較検討
- Cloudflare Pagesのセットアップ：APIトークンの権限設定で苦戦
- Account IDを直接指定して突破した瞬間
- Claude Codeが一緒にトラブルシューティングしてくれた体験

### 6. iPadでの実際の使用感（〜800字）
- Safariフルスクリーンモード（ホーム画面に追加）
- Bluetoothペダルでのスライド操作
- ダーク背景がステージ照明と干渉しない点
- 曲間のシームレスな遷移（最重要要件がちゃんと動いた）
- 残った課題や今後改善したい点

### 7. セットリスト管理とセッション保存（〜800字）
- タッチ対応ドラッグ＆ドロップ（iPad Safariで長押しで並べ替え）
- セッション管理の設計思想：複数ユーザー想定で毎回リセット
- ファイルアプリへのJSON保存/読み込み
- sessionStorageによるリロード対策

### 8. 技術的なポイントまとめ（〜800字）
- 単一HTMLファイルで完結する設計
- Web APIの活用（Wake Lock、Fullscreen、File API）
- CJK対応の自動フォントフィットアルゴリズム
- Cloudflare Pagesでの無料ホスティング＋CLIデプロイ
- コマンド1つで更新できるワークフロー

### 9. Claude Codeと作って感じたこと（〜1000字）
- 「作りたいものを言語化する力」が重要だった
- プロジェクトブリーフを書いたことが効いた
- 対話的に改善していけるのが楽しい
- トラブルシューティング（Cloudflareの権限問題）でも一緒に考えてくれた
- プログラミング経験がなくても/浅くても作れる可能性
- ただし「何が欲しいか」を明確にする必要はある

### 10. まとめ・リンク（〜400字）
- LyriFlowのURL（Cloudflare Pages）
- 同じ悩みを持つボーカリストへのメッセージ
- Claude Codeへのリンク
